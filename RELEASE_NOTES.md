# Release Policy

- **Major Version (2.x):** Significant architecture or feature changes.
- **Minor Version (1.x):** New functionality or project milestone.
- **Patch Version (1.0.x):** Bug fixes, documentation improvements, or minor enhancements.

Example:

- **1.0.0** → Initial repository
- **1.1.0** → Data Model completed
- **1.2.0** → SQL implementation completed
- **1.3.0** → Python implementation completed
- **1.4.0** → ETL completed
- **1.5.0** → Dashboards completed
- **2.0.0** → Enterprise-ready release
